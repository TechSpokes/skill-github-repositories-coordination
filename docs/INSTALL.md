# Install

Generated skills can be installed as standalone skill folders or as plugin packages.

## Installation Goal

Installation should preserve the skill directory exactly enough that `SKILL.md` can find its referenced resources.

The install docs focus on runtime artifacts because generated repositories contain more than the installed skill. Users should install release ZIP contents, not clone-time bootstrap material.

## Standalone Skill

Copy the generated skill folder from a release ZIP into a skill location your tools read. Keep `SKILL.md` and its support folders together so relative paths remain valid.

The cross-tool standard location is `.agents/skills/`. Codex scans it from the working directory up to the repository root, and GitHub Copilot also reads it. Prefer this location for a repository shared across tools.

Tool-specific locations also work:

- `.claude/skills/` for Claude Code project skills.
- `~/.claude/skills/` for personal Claude Code skills across all projects.
- `.github/skills/` for GitHub Copilot.

Rationale: Agent skills use progressive disclosure. References and assets load only when needed, but they must stay near `SKILL.md` for relative paths to work.

## Codex Plugin

Use the Codex plugin ZIP generated by the release workflow. The package contains `.codex-plugin/plugin.json` and a `skills/` directory.

## Claude Plugin

Use the Claude plugin ZIP generated by the release workflow. The package contains `.claude-plugin/plugin.json` and a `skills/` directory.
