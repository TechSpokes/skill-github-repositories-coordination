# Coordinate GitHub Repositories v1.2.0

Version `1.2.0` completes the roadmap work maintained in this repository, adds simple feedback from humans and agents, and separates delivery from external outcome evidence.

## Runtime Improvements

- Offer a sanitized feedback draft for the maintainers when a real run exposes reusable success, friction, confusion, a missing case, or unsafe guidance.
- Require only the factual observation at intake and let agents enrich optional context without inventing facts.
- Preserve exact human review and approval before public submission, and route sensitive security findings privately.
- Add a direct feedback reference while keeping the runtime portable and conversation-only.

## Roadmap Delivery

- Publish the founding portfolio case with the builder and maintainer relationship and evidence limitations intact.
- Add secure launch readiness, skill interoperability, maintenance health, and phase-by-phase delivery records.
- Separate repository delivery from adoption evidence so missing unsolicited signals limit claims without freezing authorized product work.
- Keep outside adoption, launch attention, retention, recommendation, derivative work, contribution, and succession claims explicitly unproven.

## Feedback and Evaluation

- Add a feedback form that requires only one observation from a human or agent-assisted report.
- Add feedback behavior and adversarial cases covering observation versus hypothesis, privacy, duplicate search, routing, and exact publication approval.
- Add deterministic validation for the feedback reference, public receiving surfaces, and the rule that an agent must not submit feedback automatically.

## Release Integrity

Every packaging run creates three installable ZIPs and a `SHA256SUMS` file. The release workflow generates GitHub artifact attestations for the ZIPs before uploading draft release assets.

Verify a downloaded ZIP with:

```bash
gh attestation verify ./coordinate-github-repositories-v1.2.0.zip -R TechSpokes/skill-github-repositories-coordination
```

Artifact provenance links the ZIP to its source and workflow. It does not replace content inspection or a security review.

## Install

[Download the standalone ZIP](https://github.com/TechSpokes/skill-github-repositories-coordination/releases/download/v1.2.0/coordinate-github-repositories-v1.2.0.zip), open it, and copy the complete `coordinate-github-repositories` folder into the personal skills folder for your agent.

## Package Boundary

The standalone, Codex plugin, and Claude plugin packages contain the same portable runtime core and eight focused references. They exclude repository documentation, tests, intake, local paths, private evidence, and maintenance artifacts.
