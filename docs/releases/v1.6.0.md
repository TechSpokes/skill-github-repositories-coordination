# Coordinate GitHub Repositories v1.6.0

Version `1.6.0` adds a small, progressive writing quality contract that closes issue #15 without turning repository coordination into a prose analysis system.

## Portable Runtime

- Preserve actors, actions, evidence, authority, safety boundaries, user vocabulary, and technical literals while improving coordination prose.
- Remove the fixed `This prevents ...` sentence template and explain concrete risk in natural language when it helps the user decide or reuse the rule.
- Load the new writing quality reference only after an explicit request to improve language quality, a reported clarity defect, or a requested final review of generated coordination files.
- Keep unrelated paragraph rewriting outside the skill's primary activation boundary.

## Maintainer Guidance

- Use American English for new prose while preserving quotations, product spelling, established terms, identifiers, commands, URLs, flags, and versions.
- Diagnose observable reading costs instead of treating a style pattern as proof of AI authorship.
- Rewrite overloaded sentences, dense modifier or label chains, hidden actors, stock rationale frames, redundant results, unsupported contrasts, vague claims, and unnecessary structure only when they impair meaning for the target reader.
- Revise subtractively before adding definitions, evidence limits, exceptions, relationships, or safety boundaries that are genuinely missing.

## Evaluation

- Add a 22-case applied repair corpus across runtime, security, evaluation, release, installation, and intake prose.
- Add 10 protected cases for legitimate compounds, true contrasts, commands, flags, URLs, identifiers, versions, and precise safety language.
- Add activation and behavior coverage for an explicit quality pass on generated coordination files while retaining the negative generic rewrite case.
- Validate corpus shape, coverage, unique IDs, protected literals, and applied revisions without a grammar dependency, network call, model judge, or authorship detector.

## Maintenance Budget

The v1.6.0 `SKILL.md` is one word smaller than the v1.5.0 version despite adding the optional quality pass. The optional reference contains 433 words in 51 lines and remains unloaded during ordinary runs. The maintainer instruction contains 28 lines, and deterministic validation uses only the existing Node.js process and standard library.

The detailed research archive remains in ignored private intake for later reuse and is not included in release packages. Review the public corpus only when a writing rule changes or a material defect is reported; routine patches do not require a complete style audit.

## Compatibility

This minor release preserves conversation-only use, portable frontmatter, the ten-stage coordination workflow, authority for each target, packaging formats, and existing installation paths. It adds no runtime script, external service, grammar package, required tool, or persistent profile.
