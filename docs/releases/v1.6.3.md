# Coordinate GitHub Repositories v1.6.3

Version `1.6.3` corrects the post-publication update check for releases that change repository documentation but leave the installed skill runtime unchanged.

## Observed Failure

The public v1.6.2 versionless installation passed. The contained update from v1.6.1 then reported `All skills are up to date` because both tags contain the same canonical runtime tree, but the workflow incorrectly required the installed metadata to change to v1.6.2.

The two release tags resolve `skills/coordinate-github-repositories/` to the same Git tree. GitHub CLI therefore preserved the existing runtime and its v1.6.1 source metadata instead of performing a content-free update.

## Correction

The workflow now compares the previous and current skill directory Git tree IDs before invoking GitHub CLI update. Equal tree IDs skip the content-free update and verify the existing installation against the current source while retaining the previous tag expectation.

Different tree IDs keep the existing strict path: preview and apply the contained update, then require current release metadata and current runtime content. The repository's Node.js verifier remains unchanged and platform-neutral.

## Compatibility

This patch preserves the v1.6.2 README product page, activation boundary, runtime workflow, installed references, package formats, security controls, and conversation-only fallback. It changes only release verification, documentation, and synchronized release metadata.

## Verification

Run the final release checks with the intended tag:

```bash
npm run release:preflight -- v1.6.3
```

After publication, the versionless install must resolve v1.6.3. The contained verification from v1.6.2 must detect the equal skill tree, skip the content-free update, and verify the unchanged installation against the v1.6.3 source.

Release tag: `v1.6.3`.
