# Coordinate GitHub Repositories v1.3.2

Version `1.3.2` preserves the portfolio knowledge reuse and streamlined delivery improvements from the unpublished `v1.3.1` candidate while correcting package content across hosts and adding an abandonment path supported by evidence.

## Release Correction

- Normalize packaged Markdown, JSON, YAML, and text bytes to LF before archive creation so an existing Windows checkout and a clean Linux runner package the same tagged content identically.
- Reject host-specific CRLF bytes during non-extracting archive verification.
- Snapshot Git-filtered blob identities for every tracked and nonignored untracked candidate file so the preflight represents the content Git will commit.
- Compare local preflight checksums with the draft release asset digests before publication.

## Candidate Recovery

- Treat every final-form release tag as permanently consumed.
- Allow an explicitly authorized manual workflow or immutable `abandoned/vX.Y.Z` tag to prohibit an unpublished candidate.
- Require the abandonment marker to be annotated and to resolve to the same peeled commit as the release tag.
- Recheck remote release state immediately before deleting only the matching unpublished draft.
- Refuse abandonment mutation for a published release, a missing candidate, a lightweight marker, or a marker on another commit.

## Carried Product Improvements

- Use authorized portfolio inventory to locate and evaluate familiar practices without copying them or expanding write authority.
- Build deterministic stored ZIP files without host archive commands.
- Validate the complete final candidate once, then keep tag and publication workflows focused on immutable delivery state.
- Verify the exact unpublished tag, the versionless published install, and a contained update from the previous published release.
- Keep evergreen GitHub CLI commands free of maintained project version numbers.

## Verification

Run:

```bash
npm run release:preflight -- v1.3.2
```

The local `SHA256SUMS` values must match the draft release asset digests. The tag workflow must verify the exact `v1.3.2` install. After publication, the versionless install and contained update from the previous published release must pass. The authorized abandonment workflow must mark `v1.3.1` on its original commit and remove only its unpublished draft.

Release tag: `v1.3.2`.
