# Coordinate GitHub Repositories v1.0.1

This patch makes installation clearer for people who do not use terminals or
know how Agent Skills directories are organized. Runtime coordination behavior
is unchanged.

## Install Without a Terminal

The easiest option is to copy this message into your agent:

```text
Install the public Coordinate GitHub Repositories skill globally for me from
https://github.com/TechSpokes/skill-github-repositories-coordination at
version v1.0.1. Prefer your native skill installer. Do not use GitHub's Source
code ZIP, do not overwrite an existing installation without asking, and verify
the final location.
```

Or [download the recommended standalone ZIP](https://github.com/TechSpokes/skill-github-repositories-coordination/releases/download/v1.0.1/coordinate-github-repositories-v1.0.1.zip).
Open it and copy the whole `coordinate-github-repositories` folder into your
agent's personal skills folder.

## Improvements

- Lead with a recommended personal or global install.
- Add browser-only File Explorer and Finder steps.
- Explain the three release packages in plain language.
- Warn against GitHub's automatic Source code downloads.
- Add copyable user-scope GitHub CLI commands for Codex, GitHub Copilot, and
  Claude Code.
- Show the exact final folder structure and a verification prompt.

## Package Boundary

The same portable runtime core is shipped as a standalone Agent Skill, Codex
plugin, and Claude plugin. Release packages exclude repository documentation,
intake, local paths, private research, tests, and maintenance files.
