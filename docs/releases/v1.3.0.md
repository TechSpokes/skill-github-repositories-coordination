# Coordinate GitHub Repositories v1.3.0

This release makes GitHub CLI the preferred source install and update channel while preserving the standalone, Codex plugin, and Claude plugin packages for browser installation and hosts with plugin support.

## Highlights

- Add versionless `gh skill install` guidance for Codex, GitHub Copilot, Claude Code, and capable agents without maintaining a release number in the command.
- Document the complete preview lifecycle for discovery, release resolution, pins, installed source metadata, updates, search, authentication, trust, containment, validation, and rollback.
- Keep the existing draft release workflow as the authority for curated notes, three ZIP packages, checksums, attestations, review, and publication.
- Add clean checkout `gh skill publish --dry-run` checks before packaging in CI and release automation.
- Add a platform neutral Node verifier for an already installed runtime tree.
- Add an ephemeral release event workflow that installs the latest public release and verifies its tag, repository, source path, tree identity, and runtime content without executing the skill.
- Add a root installation guide that helps a new user choose an agent, GitHub CLI, native installer, browser, project, or contributor path without editing an earlier command.
- Add a beginner guide that explains GitHub CLI, authentication, accounts, organizations, explicit repository targets, skill commands, connectors, and Windows sandbox credential visibility.
- Add a focused runtime reference so an agent asked to update this skill can identify the recorded source, check without changing files, apply only the selected update, and stop when the origin is unresolved.

## Verified Baseline

On 2026-07-18, the installed GitHub CLI v2.92.0 selected the published v1.2.0 release from the versionless command, copied the canonical runtime tree, injected source tracking metadata, refused an unapproved overwrite, reported the unpinned copy as current, skipped a pinned copy, and returned the skill through public search.

The implementation was also compared with the then-current GitHub CLI v2.96.0 source. Capability detection remains the compatibility rule because the command is in preview and its subcommands and update details continue to evolve.

Disposable update tests covered a current copy, an available update, a pinned copy, a valid copy without a lockfile, a copy without GitHub metadata, and an explicitly approved forced recovery. The tested update commands changed only the selected disposable skill and wrote tracking state only below the disposable profile.

## Release Integrity

The tag still produces the standalone skill ZIP, Codex plugin ZIP, Claude plugin ZIP, and `SHA256SUMS`. GitHub Actions attests each ZIP before upload.

The GitHub CLI source install reads the tagged Git tree rather than those ZIP assets. Publishing this reviewed release makes the same canonical runtime available through both delivery paths.

## Security and Privacy

The public install verification runs in an ephemeral runner hosted by GitHub with read-only repository permission, a temporary install directory, and a disposable user home. It compares files and metadata but never executes installed instructions or scripts.

Remote install tests must not use a maintainer's normal profile merely because `--dir` points into a sandbox. GitHub CLI also records install state below the effective user home.

## Upgrade

Install the latest release without a maintained version pin:

```bash
gh skill install TechSpokes/skill-github-repositories-coordination coordinate-github-repositories --agent codex --scope user
```

The root [installation guide](../../INSTALL.md) provides complete commands for GitHub Copilot, Claude Code, and project scope. Existing pinned installations remain pinned until explicitly unpinned.
