# Coordinate GitHub Repositories v1.4.0

Version `1.4.0` adds a short first conversation and bounded progressive discovery so a newly installed agent can learn enough to help without demanding a portfolio profile or silently expanding access.

## First Conversation

- Activate when a person asks what to do after installation or asks what the skill can help with without stating a concrete outcome.

- Explain the skill's role and limits in one sentence, then ask what made the person install it or what they want to make easier.

- Use no more than two initial follow-up questions by default before reflecting a tentative working hypothesis and offering one bounded next step.

- Skip generic onboarding when a concrete outcome is already clear.

- Preserve unknowns and avoid a broad scan when no human is available to answer.

## Progressive Discovery

- Learn from explicit statements, corrections, current decisions, and bounded authorized evidence.

- Keep confirmed facts separate from tentative inferences and reuse context that is already known.

- Offer one related optional next step by default and explain its connection to the current outcome.

- Renew relevance, evidence scope, and authority before broader discovery.

- Stop before monitoring, persistence, broader access, mutation, publication, or unrelated implementation.

## Evaluation

- Add two activation prompts for first conversation and capability orientation.

- Add behavior cases for a blank start, a first conversation about non-code work, a concrete request, declined questions, an unavailable human, later progressive discovery, a tentative relationship, and a handoff that preserves context.

- Add adversarial cases for silent portfolio profiling and repeated suggestions after dismissal.

- Register 46 cases across 14 maintained evaluation segments.
