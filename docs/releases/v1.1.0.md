# Coordinate GitHub Repositories v1.1.0

Version `1.1.0` establishes the trust, evaluation, teaching, interoperability, governance, and release foundations for the evidence-gated roadmap in issue #6.

## Runtime Improvements

- Explain why a consequential least-privilege or reversible step protects the user's work.
- Use fully qualified tool names when similar connector or MCP operations have different authorization audiences.
- Add a portable portal handoff workflow that preserves ownership, provenance, privacy, uncertainty, and graceful fallback.
- Disclose relevant maintainer commercial interests and evaluate them under the same rubric as alternatives and no change.

## Quality and Security

- Register 28 activation, behavior, and adversarial cases across 11 user and capability segments.
- Fail validation when fixtures drift from the evaluation registry or required segments lose coverage.
- Add a threat model for the untrusted-input-to-privileged-tool-to-public-output path.
- Add privacy, access-denied, ambiguous-tool, cross-repository mutation, and recommendation-conflict cases.

## Program and Governance

- Add the reviewed seven-phase roadmap with separate product and cross-track gates.
- Add records for volunteer trials, case studies, launches, model comparisons, and phase decisions.
- Add a newcomer learning path, portal positioning, contribution definition of done, maintainer model, and recommendation-independence firewall.
- Document the Windows sandbox retry path for loading an authenticated GitHub CLI keyring while preserving command-level approval and target checks.
- Make evergreen installation guidance resolve the latest tagged release without a maintained version pin.

## Release Integrity

Every packaging run creates three installable ZIPs and a `SHA256SUMS` file. The release workflow generates GitHub artifact attestations for the ZIPs before uploading the draft release assets.

Verify a downloaded ZIP with:

```bash
gh attestation verify ./coordinate-github-repositories-v1.1.0.zip -R TechSpokes/skill-github-repositories-coordination
```

Artifact provenance links the ZIP to its source and workflow. It does not replace content inspection or a security review.

## Install

[Download the standalone ZIP](https://github.com/TechSpokes/skill-github-repositories-coordination/releases/download/v1.1.0/coordinate-github-repositories-v1.1.0.zip), open it, and copy the complete `coordinate-github-repositories` folder into the personal skills folder for your agent.

## Package Boundary

The standalone, Codex plugin, and Claude plugin packages contain the same portable runtime core and seven focused references. They exclude repository documentation, tests, intake, local paths, private evidence, and maintenance artifacts.
