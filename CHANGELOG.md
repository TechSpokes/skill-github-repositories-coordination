## [v1.1.0]

- Add the evidence-gated roadmap and coordinated phase plans from issue #6.
- Add an executable evaluation registry covering activation, behavior,
  adversarial, access-denied, non-code, beginner, platform, and write-capable
  cases.
- Add a threat model for untrusted repository input, privileged tools, public
  output, privacy, and release identity.
- Teach the reason for consequential least-privilege and reversible steps.
- Add portable portal handoff guidance with fully qualified tool routing and
  graceful access fallbacks.
- Add newcomer learning, program evidence, governance, contribution review,
  recommendation-independence, and maintainer-succession contracts.
- Generate SHA-256 checksums for all release ZIPs and attest release provenance
  through GitHub Actions.
- Document the Windows sandbox retry path that lets authenticated GitHub CLI
  commands load the host keyring without weakening action-level approval.
