# Writing Quality Pass

## When to Load

Load this reference when the user explicitly asks to improve the language quality of coordination output or generated files, reports a concrete clarity defect, or requests a final language review before a handoff or publication.

Do not load it for every run. A general request to rewrite unrelated prose remains outside this skill's primary workflow.

## Goal

Improve first-pass comprehension while preserving the user's meaning, vocabulary, evidence, authority, safety boundaries, and technical literals.

## Scope

Confirm which output or generated files the pass covers. Review only that material and the minimum owning context needed to preserve meaning.

The quality pass does not authorize new repository discovery, file changes, publication, or cross-repository work. Use the authority already established for the current task.

## Preservation Checklist

Before editing, record:

- Actors, actions, objects, and ownership.
- Observations, inferences, recommendations, approvals, and verification results.
- Scope, unknowns, exceptions, privacy, and safety boundaries.
- User terms and definitions.
- Commands, flags, URLs, identifiers, versions, quotations, and established compounds that must remain exact.

## Review Sequence

1. Identify the target reader, purpose, and smallest useful output.
2. Mark any sentence whose actor, action, relationship, evidence state, or scope is unclear on the first reading.
3. Remove repetition, empty framing, obsolete alternatives, redundant result labels, decorative summaries, and unnecessary structure.
4. Split overloaded sentences and move dense labels or modifiers after the noun.
5. Replace abstract action nouns with concrete actors and verbs when responsibility matters.
6. Keep contrast only when both sides are relevant and genuinely differ.
7. Replace vague or inflated judgments with observable behavior, scope, evidence, and limitations.
8. Add words only to restore meaning that the subtractive pass exposed as missing.
9. Compare the revision with the preservation checklist and restore any lost fact or boundary.

## Protected Constructions

Do not ban hyphens, compound modifiers, passive voice, lists, long sentences, nominalizations, `rather than`, or `not ... but`. Each can be correct.

Retain conventional compounds such as `read-only`, `cross-repository`, and `least-privilege`. Do not coin a new compound label merely to shorten a revision; use an ordinary verb phrase when it is clearer. Retain a true contrast when naming the rejected alternative changes the decision. Retain passive voice when the actor is unknown, irrelevant, or intentionally withheld and the action remains clear.

## Output

Return the revised prose or apply authorized file edits. Briefly report the meaning preserved, the material defects corrected, and any construction intentionally retained because a broader rule would damage accuracy.

Do not produce an AI-authorship score or claim that a writing pattern identifies its author.
